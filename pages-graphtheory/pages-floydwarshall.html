Floyd-Warshall Algorithm

Description

Given a graph in the form of an adjacency matrix and integer representing the number of nodes in the graph, will find the shortest path distance between the every pair of nodes in the graph. It is important to note that this algorithm can handle negative edge weights but not negative cycles.

First a distance matrix of size number of nodes by size number of nodes (same size as adjacency matrix) is initalized with the content in the given adjacency matrix.

It then considers for each pair of nodes i and j, if the distance between i and j is decreased when using itermediate nodes 1 through k, the distance array representing the distance between i and j should be updated to the new shorter distance value.

The algorithm concludes when k represents all possible itermediate nodes in the graph.


Input Requirements

Takes in two command line arguments. The first argument is an integer that represents the total number of nodes in the graph.

The second argument is a string that represents all the edges between nodes in the form (first-node)/(second-node)/(connecting-edge-weight) with each group of three values separated by a comma.

Note that the main method in the program converts this input into an adjacency matrix before passing that matrix into the Floyd-Warshall Algorithm method.


Example Input and Output

Input: 4 1/2/5,1/3/8,2/3/9,2/4/2,3/4/6

Output: Shortest path between every pair of nodes using Floyd-Warshall:

Distance from node 1 to node 1: 0

Distance from node 1 to node 2: 5

Distance from node 1 to node 3: 8

Distance from node 1 to node 4: 7

Distance from node 2 to node 1: 5

Distance from node 2 to node 2: 0

Distance from node 2 to node 3: 8

Distance from node 2 to node 4: 2

Distance from node 3 to node 1: 8

Distance from node 3 to node 2: 8

Distance from node 3 to node 3: 0

Distance from node 3 to node 4: 6

Distance from node 4 to node 1: 7

Distance from node 4 to node 2: 2

Distance from node 4 to node 3: 6

Distance from node 4 to node 4: 0


Time and Space Complexity

Worse case time: O(|V|3)

Best case time: O(|V|3)

Average case time: O(|V|3)

Worse case space: O(|V|2)

Best case space: O(|V|2)

Average case space: O(|V|2)
