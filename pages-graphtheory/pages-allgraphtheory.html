All Graph Theory

Description

Given a graph in the form of an adjacency matrix, an integer representing the number of nodes in the graph, an integer representing the starting node, and an integer representing which graph theory algorithm to perform, will perform that algorithm onto the graph.

The options for graph theory algothims are: Dijkstra's Algorithm (1), Bellman-Ford Algorithm (2), and Floyd-Warshall Algorithm (3).


Input Requirements

Takes in four command line arguments. The first one being a number from 1-3 that signifies which graph theory algorithm to use.

The second argument is an integer that represents the total number of nodes in the graph.

The third argument is also an integer that represents which node is the starting node (please node that node 0 is not a valid node, the smallest possible value for this argument is 0). Also note that if you are using Floyd-Warshall you must still put a random integer to be ignored as a third command line argument even though Floyd-Warshall does not need a starting node in order for the program to compile correctly.

The fourth argument is a string that represents all the edges between nodes in the form (first-node)/(second-node)/(connecting-edge-weight) with each group of three values separated by a comma.

Note that the main method in the program converts this input into an adjacency matrix before passing that matrix into the appropriate algorithm method.


Example Input and Output

Input: 2 4 3 1/2/5,1/3/8,2/3/9,2/4/2,3/4/6

Output: Shortest path between node 3 and every other node using Bellman-Ford:

Node 1 Distance: 8

Node 2 Distance: 8

Node 3 Distance: 0

Node 4 Distance: 6
