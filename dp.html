Dynamic Programming

This page contains links to four different approaches to programming the Fibonacci Sequence from a brute-force

recursive to a Dynamic Programming solution with linear runtime and constant space complexity.

There is also a Java program that combines all four approaches into one.

Fibonacci Recursive

Description

Given an integer representing which term of the Fibonacci Sequence to compute, will compute that term of Fibonacci using brute-force unoptimized recursion.

Fibonacci Memoized

Description

Given an integer representing which term of the Fibonacci Sequence to compute, will compute that term of Fibonacci using recursion as well as having a 1-dimensional array to store computed values.

That way, if a recursive call asks to find the nth term of Fibonacci and that term has already been computed before, the recursive call can return the value stored in the array instead of having the repeatedly calculate nth term of Fibonacci again.

If a recursive call asks to find the nth term of Fibonacci and that term has not already been computed before, then the recursive call must calculate the value recursively, however can also store the value into the array in case future recursive calls asks to find that same nth term of Fibonacci.

Fibonacci Dynamic Programming

Description

Given an integer representing which term of the Fibonacci Sequence to compute, will compute that term of Fibonacci without using recursion but instead having a 1-dimensional array to store computed values.

The program first creates an array of size given integer plus 1. It then initializes the value at index 0 to 0 and the value at index 1 to 1. It then iterates though the array starting from index 2, calculate each index value by adding the values of the two previous indexes.

Once the value at the final index is computed, that value is then returned.

Fibonacci DP Space-Optimized

Description

Given an integer representing which term of the Fibonacci Sequence to compute, will compute that term of Fibonacci without using recursion but instead having a 1-dimensional array of size 2 to store computed values.

The program first creates an array of size 2. It then initializes the value at index 0 to 0 and the value at index 1 to 1. It then calculates the next term (in this case index/term 2) by adding the values at indexes 0 and 1 and stores the calculated value at index [current iteration modulo 2].

This process is then repeated until the iteration where the given integer term of Fibonacci is computed. This program works because Fibonacci only relys on the previous two terms to calculate the next term.

All Fibonacci Approaches

Description

Given an integer representing which term of the Fibonacci Sequence to compute and an integer representing which Fibonacci computional approach to take, will compute that term of Fibonacci using the selected approach.

The options for Fibonacci approaches are: brute-force recursion, memoization, standard dynamic programming, dynamic programming with space optimization.
